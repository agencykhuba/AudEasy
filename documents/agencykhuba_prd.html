Agency Khuba Product Requirements Document (PRD)

1. Project Specifics

  • Participants: Chief Leader (solo entrepreneur), SLM Central AI (orchestrator), Departments (Execution, Troubleshooting, Improvement, Security, External Audit), Stakeholders (future hires, clients).
  • Status: On target (Phase 3 MVP in progress, backend testing).
  • Target Release: Q4 2025 (full SLM workflow MVP for multi-niche apps).

2. Team Goals and Business Objectives

Automate app development across niches as a virtual DevOps team, reducing manual effort for solo entrepreneur while enabling horizontal scaling with hires. Achieve 80% automation in Phase 3 for AudEasy pilot, with SLM-driven code generation, testing, and governance.

3. Background and Strategic Fit

Why: Current Excel-based auditing (AudEasy pilot) is inefficient; Agency Khuba provides a reusable, AI-assisted SLM template for any niche, transforming manual workflows into automated, scalable solutions.

Strategic Fit: Aligns with company growth (solo to multi-client), leveraging modern AI (Phi-2) and cloud (Render.com) for horizontal scaling, with departments handled by open-source tools.

4. Assumptions

  • Technical: SLM (Phi-2) generates reliable code; PostgreSQL schema (database_schema.sql) supports UUID/ENUM for audits.
  • Business: Solo entrepreneur directs via signals; hires integrate into departments as clients grow.
  • User: Central AI "knows" past/present/future states via logs, trends, predictions.

5. User Stories

  • As a Chief Leader, I want SLM to generate code/tests from prompts, so I can initiate development without manual coding.
  • As a Troubleshooting Dept., I want pytest to validate endpoints, so backend reliability is ensured.
  • As an External Audit, I want read-only GitHub logs, so world-class oversight is possible without execution interference.
  • Success Metrics: 90% test pass rate; 50% time savings in Phase 3; scalable to 3 niches by Q1 2026. Link to spec.json for full stories.

6. User Interaction and Design

Interaction: Chief Leader inputs signals (prompts) to central AI; departments execute (e.g., SLM code gen via workflow_generate.py); feedback loops via pytest/GitHub.

Design: Modular dashboard (Confluence/Jira) for status; wireframes in schema.drawio for SLM flow.
